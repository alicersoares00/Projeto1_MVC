***Descrição do Projeto***

Este projeto foi desenvolvido com o objetivo de estudar e praticar o padrão arquitetural MVC (Model-View-Controller) em Python.
Durante o processo, decidi adicionar também um DAO (Data Access Object) para isolar a manipulação dos arquivos JSON, garantindo uma separação mais organizada entre a lógica de negócio e a persistência dos dados.
A aplicação simula um sistema de gerenciamento simples, com módulos de clientes, produtos, fornecedores, categorias, funcionários e caixa.
O módulo de Caixa registra vendas e gera relatórios (produtos mais vendidos, clientes que mais compram e total de vendas), consolidando conceitos de POO (Programação Orientada a Objetos) e boas práticas de organização de código.

***Objetivos***

- Consolidar conhecimentos em POO com Python.
- Praticar a implementação do padrão MVC.
- Explorar o uso de DAO para persistência de dados em JSON.
- Criar um sistema funcional, mas sem complexidade excessiva, focado em aprendizado.

***Tecnologias utilizadas***

- Python 3
- JSON para persistência de dados
- Estrutura baseada em MVC + DAO

***Status***

Projeto concluído como exercício de estudo.
O foco foi aprender e aplicar conceitos de backend em Python, sem a necessidade de implementar todas as funcionalidades de um sistema real (como controle de estoque).
